Como trocar sua pasta /home para outro Hd no linux mint 19.

Alexei Abianna | Developer
Mar 30 · 3 min read

Quando instalei meu linux mint 19 cinnamon para iniciar minha experiência no mundo Unix fiz apenas um simples dual boot no mesmo hd compartilhando o W10 com o Mint.

Criei uma partição para dados de 40gb apenas, como minha intenção era apenas testar, caso eu precisasse apenas instalaria novamente. Mas o tempo passou e eu transformei essa mesma instalação em meu sistema principal de desenvolvimento.

Então personalizei todo o sistema, instalei milhares de dependências do node.js, configurei meu terminal com o .zsh, enfim, estava tudo como eu queria, mas em determinado momento tive um enorme problema com o timeshift, que em outra oportunidade posso explicar para vocês como ele funciona. Então, o problema foi o de falta de espaço, logo que o sistema ficou sem espaço em disco acabei descobrindo que o linux trava e fica bem complicado de trabalhar sem pelo menos 1gb de espaço na partição a onde se encontra o sistema.

Após esse incidente que consegui resolver, foi a vez de instalar o Android Studio, o famigerado emulador para apps mobile. Sim me vi novamente em apuros quando percebi que necessitava de pelo menos 8gb para emular um smartphone em meu sistema. Foi ai que não tive mais saída tive que buscar uma alternativa para aumentar meu espaço no diretório /home.

Foi então que vasculhando no google que encontrei algumas dicas e soluções para meu problema, porém meu conhecimento em shell é bem básico. então segui os passos de um tutorial que encontrei, mas tive que ir mais afundo para resolver meu problema.

A dica seguia os seguintes passos:

  • Primeiro: No caso de uma partição nova certifique-se que a mesma esteja formatada em ext4. Caso não esteja utilize o gparted para formatar. Basta instalar executando a linha abaixo no terminal.
sudo apt-get install gparted
  • Após nossa partição estar formatada vamos listar nossos discos.
sudo fdisk -l
  • Após listar todos nosso dispositivos de disco buscamos identificar qual é o Hd que queremos mover nossa pasta /home.
  • No meu caso o disco era o sda1, o qual eu pretendia usar como /home.
  • Vamos montar a partição que iremos usar como a nova /home.
  • Crie o diretório temporário para montar a partição.
sudo mkdir /media/temp

Aqui muita atenção ao descrever qual disco.

sudo mount /dev/sda1 /media/temp
  • Mas vamos precisar do UUID da partição, para descobrir isso use o comando abaixo.
sudo tune2fs -l /dev/sda1 | grep UUID
  • Deve retornar algo como abaixo.
Filesystem UUID: 186569fa-bo26–4fc8–8d69–45czb16b01b4

Aqui será o ponto de maior atenção.

  • Com o UUID na mão vamos para a parte principal da modificação. Vamos alterar o arquivo /etc/fstab.
  • Com privilégios de administrador “root” abra o arquivo fstab com qualquer editor de texto e adicione na linha final o seguinte:
UUID=186569fa-bo26–4fc8–8d69–45czb16b01b4 /home ext4 defaults 0 2
  • E aqui a dica principal surge, lembre-se de alterar a linha com o mesmo /home logo acima no texto para apenas /. Pois isso será essencial para que o sistema identifique a ordem da montagem do sistema. Fazendo que a partição definida por você seja a nova /home
  • Agora mova sua pasta /home para a partição digitando o seguinte no terminal:
sudo mv -f /home/$USER /media/temp/
  • Espere terminar, pois pode levar alguns minutos dependendo da quantidade de arquivos na pasta.
  • Após isso fixe as permissões para o seu usuário com a linha abaixo:
sudo chown -R $USER:$USER /media/temp/$USER
  • Depois desmonte todas as partições com o comando abaixo:
sudo umount -a
  • E monte todas as partições novamente:
sudo mount -a
  • Após esses passos sua pasta já está no novo local e funcionando, reinicie o sistema para testar se tudo está ok. Caso seu sistema tenha criado uma nova pasta /home verifique o passo da edição do /etc/fstab.

Espero que isso possa te ajudar como me ajudou. Let’s code!!
